# План реализации системы Front/Back изображений

## Текущее состояние
- `Product.images` - строка JSON с массивом URL: `["url1", "url2"]`
- Генерация AI фото добавляет URL в конец массива
- Нет информации о типе изображения (front/back/side)

## Целевое состояние
- `Product.images` - строка JSON с массивом объектов:
```json
[
  {
    "url": "https://cloudinary.com/...",
    "type": "front",
    "order": 0,
    "isGenerated": false
  },
  {
    "url": "https://cloudinary.com/...",
    "type": "back",
    "order": 1,
    "isGenerated": false
  }
]
```

## Этапы реализации

### 1. Обновить Prisma схему (опционально, для документации)
- Добавить комментарии к полю `images` о новой структуре

### 2. Создать утилиты для работы с изображениями
- `fastapi_app/utils/images.py`:
  - `parse_product_images()` - парсинг с поддержкой старого и нового формата
  - `serialize_product_images()` - сериализация в новый формат
  - `add_product_image()` - добавление изображения с типом
  - `get_images_by_type()` - получение изображений по типу

### 3. Обновить API загрузки изображений
- `POST /api/v1/products/{id}/images` - новый эндпоинт
  - Параметры: `file`, `type` (front/back/side), `order`
  - Автоматическая загрузка в Cloudinary с WebP
  - Сохранение в новом формате

### 4. Обновить генерацию AI фото
- Добавить параметр `image_type` в генерацию
- Поддержка генерации back изображений на основе front референса
- Сохранение с правильным типом

### 5. Обновить существующие эндпоинты
- GET /api/v1/products - вернуть изображения в новом формате
- Обратная совместимость со старым форматом

### 6. Миграция данных
- Скрипт для конвертации существующих изображений:
  - Первое изображение = front
  - Остальные = additional
