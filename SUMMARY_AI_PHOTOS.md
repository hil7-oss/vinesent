## Что было сделано

### ✅ Проблема 1: Фотки не отображались в форме
**Причина:** Несоответствие форматов данных
- Бэкенд сохраняет: `[{url: "...", type: "...", order: 0}, ...]`
- Фронтенд ожидал: `["url1", "url2", ...]`

**Решение:**
- Исправлен парсинг в `vinesent-admin/src/app/admin/products/page.tsx`
- Исправлены утилиты `getFirstImage` и `getAllImages` в `vinesent-admin/src/lib/utils.ts`
- Теперь поддерживаются оба формата (старый и новый)

### ✅ Проблема 2: Форма не закрывалась сразу
**Причина:** Код ждал завершения фоновой генерации

**Решение:**
- Форма теперь закрывается сразу после начала генерации
- Показывает сообщение: "Генерація 6 фото запущена у фоновому режимі"
- Фотографии появляются в списке продуктов автоматически после завершения

## Как работает система

### 1. Загрузка фотографий
```
AI генерация → Cloudinary (приоритет) → Локальное хранилище (резерв)
```

### 2. Формат хранения в БД
```json
[
  {
    "url": "https://res.cloudinary.com/dkgds398y/image/upload/v1777295458/vinesent-ai/548c7ab2.webp",
    "type": "additional",
    "order": 0,
    "isGenerated": true
  }
]
```

### 3. Генерация фотографий
- **Фронт фото (обязательно):** 4 варианта
- **Заднее фото (опционально):** 2 варианта
- **Итого:** 6 фотографий

### 4. Процесс генерации
```
1. Пользователь загружает фото и нажимает "Зберегти"
2. Продукт сохраняется в БД
3. Запускается фоновая задача генерации (jobId)
4. Форма закрывается сразу
5. Backend генерирует 6 фотографий (по одной каждые 3 секунды)
6. Каждая фотография загружается в Cloudinary
7. URL добавляется в поле images продукта
8. Фотографии автоматически появляются в списке продуктов
```
