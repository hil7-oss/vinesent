Улучшение UX и представления товаров в интернет-магазине
1. Интерактивность товаров (hover / click по изображению)

Задача:
Сделать карточки товаров интерактивными.

Требования:

При наведении или клике на изображение товара:
отображается этикетка с ценой (как визуальный тег/бейдж поверх изображения)
можно добавить:
название
quick action (например: “Подробнее” / “В корзину”)
Этикетка должна выглядеть как элемент одежды (стикер/ценник), а не стандартный UI
Анимация появления — плавная (fade / scale)

Цель:
Сделать взаимодействие с товарами более живым и современным, повысить вовлечённость.

2. Онбординг / обучалка пользователя

Задача:
Добавить ненавязчивое обучение пользователя возможностям магазина.

Требования:

При первом заходе:
показать легкий onboarding (tooltip / подсказки / микроанимации)
Примеры:
“Нажми на товар, чтобы увидеть цену”
“Свайпни, чтобы посмотреть больше фото”
Не должно мешать — легко закрывается
Можно реализовать как:
guided tour
или микро-подсказки при первом взаимодействии

Цель:
Объяснить UX без перегрузки интерфейса.

3. Улучшение генерации/отображения изображений товаров

Проблема:

текущие изображения выглядят “слишком сглаженно” / неестественно
не соответствуют ожиданиям ЦА (подростки)

Задача:

либо:
улучшить качество генерации (если используется AI)
либо:
изменить пост-обработку изображений

Требования:

изображения должны:
выглядеть реалистично
сохранять текстуры ткани
не быть “переглаженными”
при генерации:
учитывать стиль ЦА (подростковая одежда)
добавить больше контраста/деталей

Цель:
Повысить доверие к товару и конверсию.
4. Отображение товара с разных сторон (Front / Back) + интеграция с генерацией изображений
Задача

Реализовать поддержку отображения товара с нескольких сторон (front / back / optional side) с возможностью генерации изображений через нейросеть на основе загруженных референсов.

Функциональные требования
4.1 Загрузка исходных изображений (админка)

Администратор должен иметь возможность:

загружать 2–3 изображения одного товара:
Front (обязательно)
Back (опционально, но желательно)
Side (опционально)

Формат:

каждое изображение помечается типом:
front
back
side
4.2 Генерация изображений через нейросеть (Nanabanana)

Текущая проблема:

генерация даёт “сглаженные”, не реалистичные изображения
теряются детали (особенно на back-дизайне)

Задача:
Использовать загруженные изображения как референсы для генерации.

4.3 Логика генерации

Backend должен:

принимать:
набор изображений (front/back)
передавать их в нейросеть как:
reference inputs

Ожидаемое поведение:

если загружено:
только front → генерируется только front
front + back → генерируются обе версии
генерация должна:
сохранять структуру одежды
сохранять декор (особенно на спине)
не “переглаживать” текстуры

4.5 Отображение на фронтенде

В карточке товара:

Реализовать переключение между состояниями:

Варианты UX:
toggle (Front / Back)
свайп (mobile)
hover (desktop → смена изображения)
мини-превью (thumbnails)
4.6 Приоритет отображения
по умолчанию показывается Front
если есть Back:
отображается переключатель
или авто-хинт (“есть дизайн сзади”)
4.7 Дополнительные UX-улучшения
бейдж:
“Design on back”
hover:
быстрый preview задней части
анимация:
плавная смена изображения
Цель
корректно отображать товары с важными деталями на задней стороне
повысить доверие к товару
увеличить конверсию за счёт лучшей визуализации
Важно для разработчика
генерация не должна быть “свободной” —
нужно максимально опираться на загруженные изображения
избегать:
сильного сглаживания
потери текстур
искажения формы одежды

5. Комплекты / наборы (outfit / set system)

Задача:
Реализовать систему комплектов одежды.

Функционал:

Один товар = “сет” (например: костюм)
Внутри:
несколько компонентов:
футболка
штаны
кепка и т.д.

Требования:

В карточке:
отображается:
общий образ (outfit)
Ниже:
список компонентов:
каждый с ценой
Возможности:
купить:
весь комплект
или отдельные элементы
UI:
“Состав комплекта”
с кликабельными элементами

Пример:

Костюм:
- Худи — 50€
- Штаны — 60€
- Футболка — 30€

Дополнительно:

при наведении на часть изображения можно подсвечивать конкретный элемент

Цель:
Увеличить средний чек и упростить выбор.

6. Отображение скрытых деталей (например: принт на спине)

Задача:
Решить проблему, когда визуально товар “простой”, но имеет важные детали.

Решения:

автоматический highlight:
“есть дизайн сзади”
бейдж:
“Design on back”
мини-preview:
при наведении показывается задняя часть

Цель:
Не терять ценность товара из-за плохой презентации.
